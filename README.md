# 🏋️ FitForge — Responsive Frontend Application

> **Task 1 | DecodeLabs Full Stack Development Internship 2026**

FitForge is a high-fidelity, responsive gym tracker web application built as a foundational **Task-01** project for the **DecodeLabs Full Stack Development Internship**. It focuses on mastering the core fundamentals of web development — HTML5, CSS3, and Vanilla JavaScript — without relying on any external frameworks.

---

## 📌 Project Overview

FitForge allows users to track gym exercises, log workouts, and monitor fitness progress with a clean, **2025/2026-compliant aesthetic** featuring a **glassmorphism dark theme**. It emphasizes the **"Mobile-First"** paradigm, ensuring a seamless interface across all screen sizes.

---

## ✨ Core Features

1. **Exercise Library** — Browse and manage exercises by muscle group and category
2. **Workout Logger** — Create and track workout sessions with sets, reps, and weight
3. **Progress Tracking** — Log body weight, measurements, and fitness milestones
4. **LocalStorage Persistence** — Data saved locally in the browser via `localStorage`
5. **Responsive Design** — Fully adaptive layout using CSS Grid and Flexbox
6. **Glassmorphism UI** — Dark theme with glass-card effects and gradient accents
7. **Form Validation** — Client-side validation to ensure mandatory data is captured
8. **Interactive Dashboard** — Summary cards with workout counts and recent activity

---

## ⚙️ Technical Implementation

- **HTML5** — Semantic landmark elements (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`)
- **CSS3** — Macro-layouts using **CSS Grid**; micro-components using **Flexbox**; CSS custom properties for theming
- **JavaScript** — Native DOM manipulation, state management via `localStorage`, event-driven architecture
- **Color Palette** — `#6C63FF` primary purple, `#0f0f1a` deep background, glassmorphism card overlays

---

## 🚀 How to Run

1. **Clone or Download** — Save the project folder to your local machine
2. **Open in Editor** — Open the project folder in **VS Code**
3. **Launch** —
   - Use the **Live Server** extension → right-click `public/index.html` → "Open with Live Server"
   - Or directly open `public/index.html` in any modern browser

> This project strictly adheres to the **"No Frameworks"** mandate, focusing on pure frontend logic and responsive architecture.
